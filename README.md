# イベントアーカイブ(サンプル)

Leaflet + 国土地理院タイルを使った、無料で公開できる静的サイトです。
イベントデータを地図と一覧表で検索・閲覧できます。バックエンド不要で GitHub Pages にそのまま公開できます。

以下がサイト
[地図サイト](https://choroniumchan-ai.github.io/sst-enzetsu-map/)

## データ構造

1件あたり次の4項目で構成されています(タイトル・カテゴリは持ちません。地名が見出しとして表示されます)。

| 項目 | 内容 |
|---|---|
| place(地名) | 見出しとして表示される地名。イベント名を含めたい場合は `本能寺の変(京都府京都市)` のように地名側に含めてください |
| date(年月日) | `yyyy` / `yyyy/MM` / `yyyy/MM/dd` のいずれかの形式。月日が不明な場合は内部的に1月1日として扱われます(並び替え・期間フィルタ用) |
| description(詳細) | 詳細説明 |
| url(リンク) | Webページへのリンク |

(`lat` / `lng` は地図表示用に別途保持しています)

## 構成

```

├── index.html          最初に読み込まれるページ
├── style.css           デザイン

└── README.md
```

## 主な機能

- キーワード検索(地名・詳細)
- 都道府県フィルター(コンボボックス。データには都道府県を持たせず、緯度経度から自動判定)
- 期間フィルタ(開始年・終了年)
- 並び替え(新しい順 / 古い順の切り替え、デフォルトは新しい順。年月日まで考慮して並びます)
- 一覧表のページング(1ページ5件、`app.js` の `PAGE_SIZE` で変更可)
- 地図の見た目切り替え(標準地図・淡色地図・白地図・写真、デフォルトは淡色地図)
- 地図と一覧表の相互連動(クリックでハイライト・パン・ポップアップ表示)
- 絞り込み結果のダウンロード(xlsx。ライブラリが読み込めない場合はJSONに自動フォールバック)

## データの編集方法

`app.js` の先頭にある `EVENTS` 配列を直接編集してください。1件はこの形式です。

```js
{ id: "11", place: "地名(またはイベント名を含む地名)", lat: 35.0, lng: 135.0,
  date: "1600/10/21",
  description: "詳細説明。",
  url: "https://example.com/" }
```

- `lat` / `lng` は10進数の緯度経度(Google マップで地点を右クリック→座標をコピー)。
- `date` は `yyyy` / `yyyy/MM` / `yyyy/MM/dd` のいずれか。月日が不明なら年だけで構いません(例: `"1457"`)。

編集したら保存するだけで反映されます。ビルドや変換の手順は不要です。

## 都道府県フィルターについて

イベントデータ自体には都道府県の情報を持たせていません。代わりに、`data/prefectures.js` に持たせた
都道府県境界データ(国土地理院「地球地図日本」を出典とする [dataofjapan/land](https://github.com/dataofjapan/land) を簡略化して使用)を使い、
各イベントの緯度経度がどの都道府県のポリゴンに含まれるかをページ読み込み時に一度だけ計算しています(`app.js` の `findPrefecture`)。

- 都道府県境界はファイルサイズを抑えるために形状を簡略化しています(誤差はおおよそ数百m〜1km程度)。海岸沿いなど境界ぎりぎりの地点では、まれに隣接県と判定される可能性があります。
- 境界のポリゴンに一致しなかった場合(海上の座標など)は、最も近い都道府県を採用するフォールバックを入れています。
- 件数が数万件規模まで増えても、判定はページ読み込み時に1回だけ行われるため、検索・絞り込み自体の速度には影響しません。

## ダウンロード機能について

一覧上部の「ダウンロード」ボタンから、**現在の検索・期間の絞り込みが反映された結果のみ**をファイルとして保存できます(ページ送りは無視して全件が対象です)。

- 形式は xlsx(Excel形式)です。詳細説明に改行が含まれていても、CSVと違ってセルの構造が崩れません。
- xlsxの生成には [SheetJS](https://sheetjs.com/) をCDN経由で利用しています(`index.html` の `xlsx.full.min.js`)。読み込めなかった場合は自動的に `events.json` としてダウンロードされます。
- 出力される列: ID, 地名, 年月日, 詳細, リンク

## GitHub Pages への公開手順

1. GitHubで新しいリポジトリを作成します(例: `japan-history-map`)。
2. このフォルダの中身一式(`index.html` がリポジトリ直下に来るように)をプッシュします。

   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```

3. GitHubのリポジトリページで `Settings → Pages` を開きます。
4. 「Build and deployment」の Source を `Deploy from a branch` にし、Branch を `main` / `/(root)` に設定して保存します。
5. しばらくすると `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

## 地図タイルについて

地図には国土地理院(GSI)の地図タイルを利用しています。無料・登録不要で使えますが、
利用にあたっては出典表示(「地図: 国土地理院」)が必要です(`app.js` に既に含めています)。

## スマホでの表示

画面幅860px以下では、地図が上・検索/一覧が下の縦積みレイアウトに自動で切り替わります。

## 件数が増えてきたら(スプレッドシート運用への切り替え)

数百〜数千件規模になり、`app.js` を直接編集するのが大変になってきたら、以前の設計に戻すと管理が楽になります。

1. スプレッドシートで `id, place, date, description, url` の列でデータ管理する
   (`sample-data/events_template.csv` がテンプレート)。
2. CSVでエクスポートし、`python3 scripts/csv_to_json.py 入力.csv data/events.json` で変換する。
3. `app.js` の `EVENTS` 配列を `fetch("data/events.json")` で読み込む形に戻す。
4. 表示件数が多くなるので、`Leaflet.markercluster`(地図側)を入れ直すことを検討してください。

このタイミングで一緒に組み直すこともできるので、件数が増えてきたら声をかけてください。
